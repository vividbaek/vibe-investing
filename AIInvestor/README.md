# AI Investor

LL 기반 텔레그램 봇. 투자 분석용 봇으로 개발되며 OpenAI 기반에서 DeepSeek 기반으로 변경하여 서비스 예정.

사유: ChatGPT는 실시간 데이터성 미반영 및 가격
특징: ChatGPT와 DeepSeek는 API 레벨이 유사하며, 딥씨크는 금융에 강점이 있음

Telegram chatbot that answers stock questions in the voice of a famous investor (default: Warren Buffett). Pulls live fundamentals via `yfinance` and routes them through DeepSeek (`deepseek-chat` / `deepseek-reasoner`) constrained by a persona system prompt. Uses DeepSeek's OpenAI-compatible Chat Completions endpoint, so the `openai` SDK is used as the client.

> Disclaimer: This bot does not provide financial advice. Output is generated by an LLM and may be incomplete or wrong. Verify before acting.

## Features

- Real-time Telegram messaging via `python-telegram-bot` v20+
- Persona-based responses (Warren Buffett, Ray Dalio, Cathie Wood; modular for more)
- Per-chat persona switching with `/persona <key>`
- yfinance-backed fundamentals: PE / forward PE / PB, ROE, margins, debt/equity, growth, 1M / 6M / 1Y price change, 52W range
- Graceful handling of invalid tickers and upstream failures
- LLM is grounded on the fetched data block to avoid hallucinated numbers
- Mandatory disclaimer line on every reply

## Setup

1. Create a Telegram bot with [@BotFather](https://t.me/BotFather) and copy the token.
2. Get a DeepSeek API key from [platform.deepseek.com/api_keys](https://platform.deepseek.com/api_keys).
3. Configure environment:

   ```bash
   cd AIInvestor
   cp .env.example .env
   # edit .env and fill in TELEGRAM_BOT_TOKEN and DEEPSEEK_API_KEY
   ```

   To use the reasoning model, set `DEEPSEEK_MODEL=deepseek-reasoner` instead of the default `deepseek-chat`.

4. Install dependencies:

   ```bash
   python -m venv .venv
   source .venv/bin/activate     # Windows: .venv\Scripts\activate
   pip install -r requirements.txt
   ```

5. Run:

   ```bash
   python main.py
   ```

## Telegram commands

| Command | Behavior |
|---|---|
| `/start` | Greeting + current persona |
| `/help` | List commands |
| `/personas` | List available personas |
| `/persona <key>` | Switch persona for this chat (e.g. `/persona dalio`) |
| _any text_ | Treated as a ticker / company name and analyzed |

## Example interaction

```
User: AAPL

Bot: [Warren Buffett on AAPL]

Apple makes the iPhone, Mac, and a sticky services business that runs on top
of an enormous installed base. It's the kind of simple, understandable
business I like — people use the product every day and pay for it again
next year.

On the fundamentals you've shown me, this is a wonderful business. Returns
on equity are extraordinary, margins are wide, and the brand is a moat you
can't easily build by spending. The question, as always, is what you pay.
A trailing PE in this range bakes in a lot of good news; if I were buying
today I'd want a margin of safety...

Recent price action — up over the year, modest pullback in the last month
— doesn't change the analysis much. We don't think in months.

Stance: a wonderful business, but I'd be patient on the entry price.

This is not financial advice.
```

## Adding a new persona

Edit `services/persona_engine.py` and add an entry to the `PERSONAS` dict:

```python
"munger": Persona(
    key="munger",
    display_name="Charlie Munger",
    system_prompt="You are Charlie Munger. Brief, blunt, mental-models...",
),
```

It will automatically show up in `/personas` and be selectable via `/persona munger`.

## Safety

- The system prompt forbids invented numbers; the LLM is told to reason only from the data block fetched from `yfinance`.
- Every persona's system prompt requires the closing line `This is not financial advice.`
- Buy / sell language is replaced with stance framing ("I'd be inclined to wait", "I'd pass", "wonderful business at a fair price").

## Known limits

- `yfinance` is unofficial and rate-limits / occasionally returns stale data; treat the snapshot as best-effort.
- The bot stores persona state per chat in process memory only — restart loses it.
- No streaming; replies are sent once the LLM call completes.
