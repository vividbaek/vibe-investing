# AI 모멘텀 기술주 듀얼 모듈 진단 프레임워크
## 멀티 LLM 투표 프로토콜 (Quality × Timing)

| 항목 | 내용 |
|---|---|
| 대상 | AI/모멘텀 기술주 (예: SNOW, 소비기반·구독형 소프트웨어) |
| 모듈 | Module 1 = 펀더멘털 품질(FQS) · Module 2 = 과열도(OHS) |
| 집계 | 다수 LLM 독립 실행 → 중앙값·최빈값·분산(CV)으로 투표 집계 |
| 기준일 예시 | 2026-05-29 (SNOW 참조 데이터 동봉) |
| 성격 | 진단·시나리오. 매매 권유 아님. 작성자는 투자자문업자가 아님 |

## 1. 배경: 왜 듀얼 모듈이고, 왜 멀티 LLM 투표인가

### 1.1 단일 점수의 함정 — 펀더멘털과 타이밍의 혼동
대부분의 종목 점수는 "좋다/나쁘다"를 하나의 숫자로 뭉갠다. 그러나 투자 판단은
서로 다른 두 질문으로 분해된다.

- **무엇을 사느냐 (기업의 질):** 마진·반복성·잉여현금흐름·희석으로 측정되는
  이익의 질. 가격과 무관하게 평가되어야 한다.
- **언제 사느냐 (진입 타이밍):** 밸류에이션·기술적 확장·심리로 측정되는 과열도.
  훌륭한 기업이라도 포물선 구간이면 추격은 자제 대상이다.

좋은 기업이 비싼 주가일 수 있고, 그 둘은 모순이 아니다. 따라서 본 프레임은
**품질(FQS)** 과 **과열도(OHS)** 를 끝까지 분리해 채점한 뒤, 마지막 단계에서만
사분면으로 결합한다.

### 1.2 단일 LLM의 함정 — "매크로는 수렴, 실행은 발산"
복수 LLM(Claude·Gemini·ChatGPT·DeepSeek 등)에 동일한 재무 데이터로 동일한
배분 과제를 주면, 거시적 방향성(레짐 판단, 정성 결론)에서는 모델 간 합의가
높게 나타나는 반면, 구체적 실행 수치(점수·배분·목표가)에서는 분산이 크게
벌어지는 경향이 관찰된다. 거시 수렴은 좁고(낮은 변동계수), 실행 발산은
넓다(높은 변동계수).

함의는 명확하다.

1. **정성 결론(사분면·등급)** 은 모델 간 최빈값으로 모으면 신뢰도가 높다.
2. **정량 수치(점수·가격 밴드)** 는 평균이 아니라 **중앙값**으로 모으고,
   **분산(CV)** 을 함께 보고해 "정밀한 척"을 차단해야 한다.
3. LLM은 엑셀이지 오라클이 아니다. 단일 모델의 한 번 출력은 표본 1개이며,
   투표는 그 표본 오차를 드러내고 줄이기 위한 장치다.

본 프로토콜은 이 발견을 집계 규칙(섹션 4)에 직접 반영한다.

## 2. 분석의 틀

### 2.1 두 개의 축
| 축 | 약어 | 방향 | 의미 |
|---|---|---|---|
| 펀더멘털 품질 | FQS (0~100) | 높을수록 우수 | 이익의 질. "무엇을 사느냐" |
| 과열도 | OHS (0~100) | 높을수록 과열 | 진입 위험. "언제 사느냐" |

두 점수는 **방향이 반대**다. 내구성이 강한 기업은 FQS가 높아지는 동시에 OHS의
내구성 항목(역가중)이 과열 점수를 깎는다 — 즉 펀더멘털은 과열의 완충재다.

### 2.2 사분면 매트릭스 (결합은 마지막에 단 한 번)
임계값: FQS 65(B등급 경계), OHS 56(Trim-Wait 경계).

| | OHS < 56 (냉각) | OHS ≥ 56 (과열) |
|---|---|---|
| **FQS ≥ 65 (고품질)** | ① Accumulate — 분할진입 적합 | ② Hold / Trim-Wait — 소유가치 O, 추격 X |
| **FQS < 65 (저품질)** | ③ Avoid / Value-Trap 관찰 | ④ Overheated Speculation — 회피 |

- ① **Accumulate:** 질 좋은 기업이 냉각 구간. 규율적 분할 진입에 가장 적합.
- ② **Hold / Trim-Wait:** 질은 충분하나 가격이 분출. 보유 가치는 인정하되
  신규 추격은 자제, 눌림 대기. (현재 SNOW가 여기 — 섹션 6)
- ③ **Avoid / Value-Trap:** 싸 보여도 질이 낮음. 모멘텀도 없어 함정 위험.
- ④ **Overheated Speculation:** 질도 낮은데 과열. 추격은 투기. 회피.

### 2.3 멀티 LLM 투표의 역할
각 LLM은 사분면(정성)과 FQS·OHS·가격 밴드(정량)를 **독립적으로** 산출한다.
집계기는 사분면을 최빈값으로 확정하고, 수치는 중앙값으로 모은 뒤 분산을 보고한다.
사분면 합의가 강하면(분산 작음) 신뢰도 상향, 수치 분산이 크면 가격 밴드를
넓은 시나리오로만 취급한다.

## 3. 배포용 프롬프트 (각 LLM이 독립 실행)

> 아래 블록을 Claude·Gemini·ChatGPT·DeepSeek 등 각 모델에 **동일하게** 투입한다.
> INPUT은 1차 데이터(10-Q/10-K/8-K/IR)로 교체한다. SNOW용은 섹션 5 참조 데이터를
> INPUT에 붙여 즉시 실행 가능.

```text
# ROLE
당신은 규율 기반 기술주 퀀트 애널리스트다. 임무는 매수 권유가 아니라, 주어진
기업을 두 모듈로 독립 채점하는 것이다 — Module 1: 펀더멘털 품질(FQS),
Module 2: 과열도(OHS). 마지막에만 사분면으로 결합한다.

# CORE PRINCIPLES
1. 무엇(품질) vs 언제(타이밍)를 끝까지 분리한다. 좋은 기업이 비싼 주가일 수 있다.
2. 매출 성장률보다 이익의 질(마진·반복성·잉여현금흐름·희석)을 우선한다.
3. GAAP과 비GAAP을 분리 검증한다. 비GAAP만 보고 판정하지 않으며, 격차의
   주성분(주식보상비용 SBC)을 명시적으로 추적한다.
4. 레버리지/인버스 ETF는 일일 리셋 decay가 있는 단기 전용 수단으로만 다룬다.
5. 모든 수치는 입력된 1차 데이터에 근거한다. 추정 시 [추정] + 근거를 붙이고,
   알 수 없으면 '데이터 없음'으로 둔다. 수치를 지어내지 않는다.
6. 당신은 투자자문업자가 아니다. 출력은 진단이며 권유가 아니다.

# INPUT
[종목/기준일/현재가] : {...}
[비즈니스 모델]      : {구독/소비기반/혼합}
-- 펀더멘털 --
제품(핵심)매출/YoY, 최근 4분기 YoY 추이, 반복매출 비중
매출총이익률(GAAP/비GAAP), 영업이익률(GAAP/비GAAP), Rule of 40
NRR, RPO/백로그/YoY, $1M+ 고객 수·집중도
영업현금흐름/FCF/FCF마진, FCF 대비 SBC
SBC(매출 대비 %), 가중평균 주식수 YoY, 자사주/M&A
경쟁구도, GAAP-비GAAP 격차 주성분, 이연매출/일회성, 가이던스 상향/하향
-- 타이밍 --
Fwd P/S(또는 Fwd P/E), 동지표 5년 백분위
RSI(14), 200일 이평 이격(%), 최근 실적 갭(%)
30일 목표가 상향 건수, 공매도 비중(%)
ARDS Phase (1 Expansion / 2 Late-Cycle / 3 Recession-Warning / 4 Recession)

# MODULE 1 — FUNDAMENTAL QUALITY SCORE (FQS, 높을수록 우수)
| 축 | 가중 | 핵심 질문 | 레드플래그 |
| A 성장의 질 | 15% | 가속/감속, 유기적인가 | M&A로 부풀린 성장, 둔화 |
| B 마진·Rule40 | 20% | GAAP 흑자 경로가 있나 | 비GAAP만 흑자, GAAP 적자 고착 |
| C 반복성·가시성 | 20% | NRR>110%·RPO 가속인가 | NRR 압축, RPO 둔화, 고객 집중 |
| D 현금흐름의 질 | 15% | FCF가 SBC를 압도하나 | SBC로 상쇄되는 회계적 FCF |
| E 희석·자본배분 | 15% | 주식수 증가가 통제되나 | SBC 과다·지속 희석, 무분별 M&A |
| F 해자·경쟁 | 10% | 전환비용·점유 우위 | 가격경쟁 노출, 대체재 위협 |
| G 회계 품질 | 5% | 일회성·이연으로 가린 부분 | 발생액 급증, 비GAAP 정의 변경 |
각 축 0~100 채점 → 가중합 = FQS. 등급: 80+ A / 65~79 B / 50~64 C / <50 D.

# MODULE 2 — OVERHEATING SCORE (OHS, 높을수록 과열)
| 성분 | 가중 | 채점 기준(높을수록 과열) |
| A 밸류에이션 | 20% | 배수·5년 백분위가 고점권일수록 ↑ |
| B 기술적 | 30% | RSI·이평 이격·실적 갭이 클수록 ↑ |
| C 포지셔닝 | 20% | 상향 군집·심리 과열일수록 ↑ |
| D 내구성(역가중) | 15% | 펀더멘털이 강할수록 점수 ↓ (과열 완충) |
| E 매크로(ARDS) | 15% | Late-Cycle/Recession-Warning일수록 ↑ |
각 성분 0~100 채점 → 가중합 = OHS.
Phase: <36 Accumulate / 36~55 Hold / 56~75 Trim-Wait / 76+ Overheated.

# 사분면 결합 (임계값 FQS 65, OHS 56)
FQS≥65 & OHS<56 → Accumulate
FQS≥65 & OHS≥56 → Hold/Trim-Wait
FQS<65 & OHS<56 → Avoid/Value-Trap
FQS<65 & OHS≥56 → Overheated-Speculation

# 인스트루먼트 매핑 (기술주 전용)
- 단일주 레버리지/인버스: 실재 ETF 티커만. 없으면 '부재 → 섹터/지수 대체'.
- 섹터·테마 롱 ETF: {티커}
- 지수 레버리지/인버스(헤지): {티커}
- 운용 규칙: 레버리지·인버스는 30일 이내 청산, 횡보 시 즉시 정리(일일 리셋 decay).
```

## 4. 투표·집계 프로토콜

N개 모델의 JSON을 모은 뒤 다음 규칙으로 집계한다. (수동·코드 모두 가능)

| 대상 | 집계 방식 | 근거 |
|---|---|---|
| 사분면(quadrant) | **최빈값** | 정성 결론은 매크로 수렴 → 합의 강함 |
| FQS / OHS | **중앙값** | 실행 발산 → 평균은 이상치에 취약 |
| 서브스코어 | 중앙값 + 범위(min~max) | 모델 간 견해차 노출 |
| 가격 밴드 | bull/base/bear 각 **중앙값**, 폭은 min~max | 분산 클수록 신뢰 낮음 |
| confidence | 평균 × 합의보정 | 아래 규칙 |

**분산 측정(CV = 표준편차 ÷ 평균):**
- FQS·OHS의 CV가 낮으면(예: <10%) 수치 신뢰 상향.
- 가격 밴드 중심값의 CV가 높으면(예: >25%) 밴드를 "넓은 시나리오"로만 사용.

**합의 신뢰도 규칙:**
- 사분면 만장일치 → 최종 confidence ×1.15 (상한 0.95)
- 사분면 분열(최빈값 점유율 <60%) → confidence ×0.7, "판정 보류·재실행" 표기.

**집계기 프롬프트 (별도 1회):**
```text
아래 N개 JSON은 동일 종목에 대한 독립 LLM 진단이다. 다음을 산출하라:
1) quadrant 최빈값과 점유율
2) FQS·OHS 중앙값과 CV
3) 서브스코어 중앙값·범위 표
4) 가격 밴드 bull/base/bear 중앙값과 폭(min~max), 중심값 CV
5) 합의 신뢰도 규칙 적용한 최종 confidence
6) 모델 간 '가장 크게 갈린 항목' 3개 (발산 진단)
7) 한 줄 종합: 펀더멘털 vs 타이밍을 분리한 결론
수치를 새로 지어내지 말고 입력 JSON에서만 집계하라. 면책 문구를 말미에 둘 것.
[여기에 N개 JSON 붙여넣기]
```

## 7. 한계 및 면책

- 점수·가중치·임계값(FQS 65, OHS 56)은 작성자가 구성한 채점 규칙이며 주관적
  판단을 포함한다. 가중 스킴을 바꾸면 사분면 경계에서 판정이 이동할 수 있다.
- [추정] 항목(Fwd P/S, 5년 백분위, 목표가 상향 건수, SBC, ARDS Phase 등)은
  1차 수치 확보 시 교정해야 정밀도가 오른다.
- 단일주 인버스 ETF는 실재 미확인이며 임의 티커를 기재하지 않았다.
- 가격 밴드는 예측이 아니라 시나리오다. 멀티 LLM 투표는 표본 오차를 줄이는
  장치일 뿐, 미래를 보장하지 않는다. LLM은 엑셀이지 오라클이 아니다.
- 본 문서는 펀더멘털·타이밍 진단이며 매수·매도 권유가 아니다. 작성자는
  투자자문업자가 아니다. 모든 투자 판단과 책임은 본인에게 있다.
